Вывод модулей

Вывод модулей работает через базовую функцию module(), где в качестве аргумента указывается массив из значений:
	- название нужного модуля
	- параметр модуля (как правило, это название папки внутри модуля)
	- шаблон, в котором модуль будет выводится
	- изменение настроек модуля
	
Вторым аргументом задается значение this, которое также может использоваться в обработчике модуля

Если при вызове функции не указан параметр модуля, то он заменяется на default
Если при вызове функции не указан шаблон, то проверяется, есть ли шаблон с тем же названием, что и параметр, и если есть, то подгружается он, если нет - то шаблон default (по-умолчанию)

Четвертый параметр должен идти как строка в формате json. Если при вызове функции указаны изменения настроек, то эти настройки будут добавлены поверх настроек модуля после их прочтения.
Это дает возможность слегка кастомизировать параметры модуля в зависимости от внешних условий. Например, выводить соответствующие меню одного и того же типа из разных статей.

После вызова модуля, происходит формирование его структуры, которая выглядит так:

$module
	name - название модуля (из функции)
	param - параметр модуля (из функции)
	template - шаблон (из функции)
	path - абсолютный путь внутрь папки вызванного модуля
	this - дополнительное значение при вызове модуля
	settings - настройки модуля
	data - данные модуля
	var - массив с переменными, удобен тем, что переменные можно хранить внутри объекта для каждого модуля, после вызова модуля, они сами очищаются

Настройки всегда разные для каждого модуля.

Настройки модуля формируются двумя способами:
	из файла settings.ini, который должен лежать в папке данных модуля
	если файл settings.ini содержит в первом уровне структуры название параметра модуля, то настройки сначала читаются оттуда
	если файл settings.ini содержит в первом уровне структуры название 'default', то оттуда читаются настройки по-умолчанию
	если файл settings.ini не содержит в первом уровне структуры ни того, ни другого, то считается, что он и является настройками по-умолчанию
	(поэтому если вы все настройки всех шаблонов помещаете в settings, обязательно задавайте в нем структуру default)
	из файла .ini, который назван так же, как параметр модуля
	из четвертого параметра вызова модуля
	(причем эти настройки считаются приоритетными, поэтому они будут заменять любые другие настройки модуля)

Примеры тождественных настроек для вызова модуля с параметром 'first':

	first.ini
		value1 : 1
		value2 : 2
		value3 : 3

	default.ini
		value0 : 0

	settings.ini
		first
			value1 : 1
			value2 : 2
			value3 : 3
		default
			value0 : 0
